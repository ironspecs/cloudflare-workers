Local Example

One worker, two UI options.

This page demonstrates the two supported browser integration paths: a worker-managed dialog layer, and a low-level browser SDK for teams that want to own the UI themselves.

See Template Mode
When served from localhost or 127.0.0.1, the worker still validates the full flow but returns SINK_ACCEPTED instead of storing the address.

Option A: Browser SDK

<script src="/newsletters.js" defer></script>
<div id="turnstile"></div>
<script>
const session = await window.Newsletters.createSession({ listName: 'weekly' });
const turnstile = await window.Newsletters.renderTurnstile('#turnstile', {
  siteKey: session.siteKey,
});

const result = await window.Newsletters.subscribe({
  email: 'person@example.com',
  listName: 'weekly',
  submitToken: session.submitToken,
  turnstileToken: turnstile.getToken(),
});
</script>

Option B: Script-Selected Template

<script
  src="/newsletters.js?template=starter"
  defer
></script>
<script>
document.getElementById('newsletter-signup').addEventListener('click', () => {
  window.Newsletters.open({ listName: 'weekly' });
});
</script>

The high-level template is selected once in the script URL. The worker preloads only that template, and open() uses it automatically.
